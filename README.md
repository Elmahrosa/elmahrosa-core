## üõ† Core Functions

### 1. Registry Management
Every project in the ecosystem must be registered in the `ecosystem-map.json`. This ensures that dependencies (like Identity and Payments) are correctly linked across borders.

### 2. Institutional Governance
This repo hosts the **Global TEOS Constitution**. Every sub-project pulls governance rules from here to ensure that "Code is Law" and that law is compliant with both local and global regulations.

### 3. Centralized Audit Aggregation
While each repo maintains its own `/audit` logs, `elmahrosa-core` provides the tooling to aggregate these logs into a single, immutable proof of transparency for regulators and international partners.

## üöÄ How to Add a New "Spoke"
To link a new repository to the ecosystem:
1. Fork this repo.
2. Add your repository details to `ecosystem-map.json`.
3. Ensure your repo contains a `teos-manifest.json`.
4. Submit a Pull Request for governance review.
